# Apple Time to Leave 功能说明

## 📅 更新日期：2025-10-03

## 🎯 功能概述

为所有现场服事事件添加**Apple Time to Leave（出发时间提醒）**功能，让iPhone和Mac的日历应用能够根据实时交通状况自动计算并提醒最佳出发时间。

## ✨ 什么是Time to Leave？

**Time to Leave**是Apple日历的智能提醒功能，它能够：

1. 📍 **自动定位** - 检测您当前的位置
2. 🚗 **路线计算** - 计算到达目的地的路程和时间
3. 🚦 **实时路况** - 考虑当前的交通状况
4. ⏰ **智能提醒** - 在应该出发的时间提醒您
5. 🔄 **动态调整** - 路况变化时自动调整提醒时间

## 🎨 功能特性

### 为什么有用？

**场景示例**：
```
靖铮的11/09主日音控服事
├─ 到场时间：09:00
├─ 地点：200 Cultivate, Irvine, CA 92618
└─ 路程：根据您的位置自动计算

Apple日历会：
✅ 自动检测您在哪里（家里、公司、或其他地方）
✅ 计算到教会的路程时间（例如25分钟）
✅ 考虑当前路况（如果堵车，可能需要35分钟）
✅ 在应该出发的时间提醒您（例如8:25或8:15）
```

### 三重提醒系统

现在每个现场服事事件有**三个提醒**：

| 提醒类型 | 触发时间 | 说明 |
|----------|----------|------|
| **🚗 Time to Leave** | 智能计算 | Apple根据路况自动提醒 |
| **⏰ 提前30分钟** | 08:30 (9:00服事) | 固定提前提醒 |
| **🔔 开始时** | 09:00 (9:00服事) | 到场时间提醒 |

## 🔧 技术实现

### ICS文件中的关键属性

#### 1. 事件级别属性
```ics
X-APPLE-TRAVEL-ADVISORY-BEHAVIOR:AUTOMATIC
```
这个属性告诉Apple日历启用智能出行建议。

#### 2. 特殊的VALARM配置
```ics
BEGIN:VALARM
ACTION:DISPLAY
DESCRIPTION:Time to Leave - 音控
TRIGGER:PT0M
X-APPLE-DEFAULT-ALARM:TRUE
X-APPLE-PROXIMITY:DEPART
END:VALARM
```

**属性说明**：
- `TRIGGER:PT0M` - 触发器设为0，Apple会自动计算
- `X-APPLE-DEFAULT-ALARM:TRUE` - 标记为Apple默认提醒
- `X-APPLE-PROXIMITY:DEPART` - 设置为离开提醒模式

### 代码实现

```python
# 添加Apple Travel Time提醒功能
event.add('X-APPLE-TRAVEL-ADVISORY-BEHAVIOR', 'AUTOMATIC')

# Time to Leave提醒
alarm_travel = Alarm()
alarm_travel.add('action', 'DISPLAY')
alarm_travel.add('description', f'Time to Leave - {role}')
alarm_travel.add('trigger', timedelta(minutes=0))
alarm_travel.add('X-APPLE-DEFAULT-ALARM', 'TRUE')
alarm_travel.add('X-APPLE-PROXIMITY', 'DEPART')
event.add_component(alarm_travel)
```

## 📱 使用说明

### 在iPhone上使用

1. **导入日历**
   - 打开邮件或Safari中的ICS文件链接
   - 点击"添加到日历"
   - 或订阅ICS URL

2. **启用Time to Leave**
   - 日历会自动识别Time to Leave提醒
   - 确保已开启**定位服务**
   - 确保已开启**日历的定位权限**

3. **查看提醒**
   ```
   设置 → 隐私 → 定位服务 → 日历 → "使用App期间"
   ```

4. **接收提醒**
   - 系统会在计算出的出发时间发送通知
   - 通知会显示预计路程时间
   - 点击通知可查看路线

### 在Mac上使用

1. **导入日历**
   - 双击ICS文件
   - 或在日历应用中导入

2. **查看Time to Leave**
   - 打开事件详情
   - 会看到"出发时间"建议
   - 显示预计路程时间

3. **同步到iPhone**
   - 通过iCloud自动同步
   - iPhone上也会收到Time to Leave提醒

## ✅ 优势对比

### 传统固定提醒 vs Time to Leave

| 对比项 | 固定提醒 | Time to Leave |
|--------|----------|---------------|
| **提醒时间** | 固定（如8:30） | 智能计算 |
| **考虑位置** | ❌ 不考虑 | ✅ 自动检测 |
| **考虑路况** | ❌ 不考虑 | ✅ 实时路况 |
| **动态调整** | ❌ 不调整 | ✅ 自动调整 |
| **准确性** | ⚠️ 可能过早或过晚 | ✅ 刚刚好 |
| **用户体验** | 一般 | 极佳 |

### 为什么需要保留固定提醒？

虽然有Time to Leave，我们仍然保留两个固定提醒：

1. **提前30分钟** - 作为备用，即使不开启定位也有提醒
2. **开始时** - 提醒服事已经开始，如果还没到要抓紧

这样提供**三重保障**，确保不会错过服事！

## 🔒 隐私和权限

### 需要的权限

1. **定位服务**
   ```
   设置 → 隐私 → 定位服务 → 开启
   ```

2. **日历的定位权限**
   ```
   设置 → 隐私 → 定位服务 → 日历 → "使用App期间"
   ```

### 隐私说明

- ✅ Time to Leave是本地计算，不会上传您的位置
- ✅ 只在有事件时才使用定位
- ✅ 不会持续追踪您的位置
- ✅ 可以随时关闭定位权限（会失去Time to Leave功能）

## 📊 影响范围

### 受影响的事件

**只有现场服事事件才有Time to Leave**：

✅ 有Time to Leave：
- 音控服事
- ProPresenter播放服事
- 摄像导播服事

❌ 没有Time to Leave：
- ProPresenter更新（远程工作）
- 视频剪辑（远程工作）

### 不受影响的功能

- ✅ 远程工作事件（ProPresenter更新、视频剪辑）
- ✅ 其他平台的日历订阅
- ✅ API接口
- ✅ 前端页面

## 💡 使用建议

### 最佳实践

1. **确保开启定位**
   - 否则Time to Leave无法工作
   - 会退化为固定提醒

2. **提前查看路线**
   - 点击事件可以看到到教会的路线
   - 可以选择不同的交通方式

3. **留有缓冲时间**
   - Time to Leave计算的是预计到达时间
   - 建议提前5-10分钟出发

4. **检查提醒设置**
   ```
   设置 → 通知 → 日历 → 允许通知
   ```

### 常见问题

**Q1: 为什么没有收到Time to Leave提醒？**

A: 检查以下几点：
1. 定位服务是否开启
2. 日历的定位权限是否为"使用App期间"
3. 事件是否在未来（过去的事件不会提醒）
4. 事件是否有正确的地点信息

**Q2: Time to Leave提醒时间为什么变了？**

A: 这是正常的！Time to Leave会根据：
- 您的当前位置
- 实时路况
- 交通方式

动态计算最佳出发时间。

**Q3: 我不想用Time to Leave怎么办？**

A: 两种方法：
1. 关闭日历的定位权限（仍会有固定提醒）
2. 在事件中删除Time to Leave提醒（保留其他提醒）

**Q4: 其他人看得到我的位置吗？**

A: **不会**！Time to Leave是本地功能，只在您的设备上计算，不会分享给任何人。
